## P0（本周必做 — 直接影响稳定性）

### P0-1 独立 Evaluator Agent 验收闭环

**对应层**：5. 观测与独立评估 + 6. 约束校验

**现状**：Agent 自评输出质量，缺乏独立验收。云枢只负责拆任务分派，不验证结果。

**要做**：
- 新增 Evaluator Agent 类型（独立 Hermes profile）
- 云枢拆任务时自动配 Evaluator（Planner → Generator → Evaluator 三角）
- Evaluator 在隔离沙箱里对 Generator 产出做真实环境测试
- 验收不通过 → 自动回写问题 → Generator 重试修正 → 最多 3 轮
- 验收标准写入系统规则（格式校验 / 文件存在性 / 代码可运行 / API 可达）

**涉及文件**：`yunshu_io.py`（编排层），新增 `evaluator_profile/` Hermes profile

---

### P0-2 上下文 Reset（长任务交接）

**对应层**：1. 上下文边界管控

**现状**：长链路任务跑到一半，上下文窗口爆炸，Agent 开始遗忘早期约束、重复操作、或直接跑偏。

**要做**：
- 超过 80% 上下文窗口时触发 Reset
- 保存中间产物到临时文件（JSON checkpoint）
- 创建新 Agent 子进程，读取 checkpoint 继续执行
- 父 Agent 只保留"当前任务目标 + 最近一步结果"
- 新 Agent 系统提示写入"你正在继续一个中断的任务，已完成步骤：..."

**涉及文件**：`yunshu_io.py`，`redis_worker.py`

---

### P0-3 工具调用时机/频率管控

**对应层**：2. 工具调度系统

**现状**：工具全开着，Agent 可以随时调用任意工具。长任务中会出现过度调用（刷 terminal）或遗漏调用（该搜不搜）。

**要做**：
- 每个子任务声明所需工具集（不要全量挂载）
- 工具调用频率限制：30秒内最多 N 次同类型调用
- 超限 → 注入系统提示"请总结当前结果再决定下一步"
- 任务类型自动匹配工具集（代码任务 → terminal+file / 研究任务 → web / 文档任务 → feishu_doc）

**涉及文件**：`yunshu_io.py`（工具分配逻辑）

---

## P1（下周 — 体验与可靠性提升）

### P1-1 用户偏好持久化层

**对应层**：4. 记忆与状态管理

**要做**：
- 从 KnowledgeEntry 中拆分 UserPreference 模型
- 字段：输出格式 / 语言 / 详细程度 / 通知偏好 / 快捷键
- Agent 每次对话启动时自动预加载用户偏好
- 偏好变更时自动更新（不再每次问"你喜欢什么格式"）

---

### P1-2 信息分区供给机制

**对应层**：1. 上下文边界管控

**要做**：
- 系统提示拆分为四区：规则区 / 任务区 / 状态区 / 证据区
- 每区独立加载，Agent 不能修改规则区
- 证据区自动裁剪（只保留高相关性文档摘要）

---

### P1-3 故障自动回滚

**对应层**：6. 约束校验与故障恢复

**要做**：
- 操作前快照工作目录（tar 备份）
- 操作失败 → 回滚到快照 + 重试
- 连续 3 次失败 → 降级为纯文本回复 + 通知用户
- 沙箱超时 → 保存中间产物 + 标记任务"需续跑"

---

## P2（两周后 — 规模化）

### P2-1 独立验收报告

**要做**：Evaluator 验收完成后输出结构化报告（通过项 / 失败项 / 建议 / 评分），前端用 ScoreBadge 渲染

### P2-2 任务执行可视化时间线

**要做**：每个子任务记录开始/结束时间、工具调用次数、上下文利用率，前端用 Timeline 组件展示

### P2-3 系统规则自动治理

**要做**：工程师沉淀的规则写入 Rule 模型 → Agent 执行前自动加载 → 执行后自动更新规则生效统计 → 低效规则自动标记废弃

---

## 总结

| 优先级 | 数量 | 预估工时 | 目标 |
|--------|------|----------|------|
| P0 | 3 | 3-5 天 | 长链路任务不跑偏、产出可验收 |
| P1 | 3 | 3-5 天 | 体验提升、故障自愈 |
| P2 | 3 | 3-5 天 | 规模化运营 |

**核心思路**：不做模型层改动，只补 Harness 层管控。Anthropic/OpenAI/Long 的实践证明，这一层迭代对稳定性的提升远超模型升级。
