## 二、三级漏斗筛选策略（替代原方案的单层判断）

### 第一级：规则引擎硬过滤（秒级，零成本）

RPA 抓取到简历文本后，**先不调用 AI**，用规则引擎秒筛：

| 规则项 | 判断方式 | 动作 |
|--------|---------|------|
| 学历不匹配 | 关键词匹配（大专/本科/硕士） | 直接淘汰 |
| 工作地点不符 | 城市匹配 | 直接淘汰 |
| 期望薪资超预算 | 数字区间匹配 | 直接淘汰 |
| 核心技能缺失 | JD关键词命中率 < 30% | 直接淘汰 |

> **效果：** 过滤掉 40-50% 的无效简历，只把有意义的送 AI，**省 token 也省时间**。

### 第二级：Agent 置信度评估（RPA 截图，中等成本）

通过第一级过滤的简历，截图送 AI 做深度匹配评估：

- 输入：RPA 分区域截图（基本信息 + 工作经历 + 项目经验，各自独立截）
- AI 对比 JD 结构化画像（技能、年限、行业等）
- 输出：置信度分数 + 各维度评分

**分级动作：**
| 置信度 | 动作 |
|--------|------|
| > 85% | 进入第三级深度评估 + 尝试下载附件简历 |
| 70% - 85% | 标记"建议人工复核"，同步飞书但不触发自动化 |
| < 70% | 淘汰，记录原因 |

### 第三级：深度评估（附件简历，低频高价值）

仅对置信度 >85% 的候选人触发：
- RPA 下载附件简历 PDF（频率低，安全）
- AI 解析 PDF 做更细粒度评估
- 输出 S/A/B 等级

## 三、RPA 截图优化（纯客户端，零风险）

| 优化点 | 具体做法 |
|--------|---------|
| **分区域截图** | 基本信息区、工作经历区、项目经验区各自独立截图，不截整页长图 |
| **渲染等待** | 翻到简历页后 sleep 1-2 秒，等异步内容加载完 |
| **固定窗口** | RPA 保持固定分辨率/窗口大小，保证每次截图的像素布局一致 |
| **优先抓 DOM** | 如果 RPA 工具支持提取页面元素文本，直接抓 DOM 文本代替 OCR |

## 四、JD 结构化画像（匹配基准）

原方案缺少"匹配基准"——AI 必须知道在跟什么岗位比。

**实现方式：** 发布岗位时，人工（或 AI）将 JD 拆成结构化字段：

```
岗位画像（Python后端工程师）:
├── 硬性条件
│   ├── 学历: 本科及以上
│   ├── 年限: 3年以上
│   └── 地点: 北京
├── 技能要求
│   ├── 必须: [Python, Django/Flask, MySQL, Redis]
│   └── 加分: [Docker, K8s, 消息队列]
└── 行业偏好: [互联网, SaaS]
```

Agent 做的是"简历 vs 岗位画像"的对比匹配，不是凭空判断。

## 五、评估结果触发自动化动作

原方案的评估表填完就完了。改为**评估结果直接驱动行为**：

| 评估结果 | 触发动作 |
|---------|---------|
| S 级（>90%） | 自动飞书拉群（HR+面试官）+ 推送候选人名片 |
| A 级（85-90%） | 推送到 HRBP 飞书消息 + 标记优先联系 |
| B 级（70-85%） | 同步评估表，不触发通知，HR 自主查看 |
| 空窗期高风险 | 标注"面试时重点询问"，不淘汰 |
| 稳定性预测低 | 降一级处理 |

## 六、时间维度和过程追踪

每个候选人从进入流程开始就打上时间戳和标签：

| 追踪字段 | 用途 |
|---------|------|
| 首次发现时间 | 判断是否重复曝光 |
| 渠道来源 | 巡检 vs 牛人市场，后期能算出渠道 ROI |
| 当前状态 + 停滞时长 | 超过 48h 未处理自动升级提醒 |
| 历史判定记录 | 同一候选人多次出现时的参考 |

有了这些数据，「仪表盘」和「每日复盘」才有真正的内容可看。

## 七、BOSSS 账号安全策略

| 策略 | 具体措施 |
|------|---------|
| **渐进式下载** | 仅对 >85% 的候选人下载附件（占总数的 5-10%） |
| **操作间隔** | RPA 每次操作间隔 3-5 秒，模拟真人速度 |
| **分时段运行** | 避开 BOSS 高峰期（上午 9-11 点），下午/晚上跑 |
| **每日上限** | 单日下载不超过 10-20 份 |
| **账号轮换** | 如果有多账号，可轮换降低单账号风险 |

## 八、实施优先级

| 优先级 | 模块 | 理由 | 依赖 |
|--------|------|------|------|
| 🔴 P0 | 分区域截图 + 渲染等待 | 提升 AI 判断准确率的基础 | 仅 RPA 配置 |
| 🔴 P0 | 规则引擎硬过滤 | 省 AI token 成本 40%+ | 需整理 JD 硬性条件 |
| 🟡 P1 | JD 结构化画像 + Agent 置信度 | 核心匹配逻辑 | P0 完成后 |
| 🟡 P1 | 评估结果触发自动动作 | 从"填表"到"行动" | P1 Agent 就绪 |
| 🟢 P2 | 渐进式附件下载 + 深度评估 | 提升高价值候选人判断 | P1 验证后 |
| 🟢 P2 | 时间追踪 + 渠道标签 | 数据看板有内容可看 | 贯穿全流程 |
| ⚪ P3 | RPA 抓 DOM 文本替代 OCR | 进一步提效 | 需验证 RPA 工具能力 |
| ⚪ P3 | 每日复盘定时推送 | 持续优化 | 有数据积累后 |

## 九、与原方案对比

| 维度 | 原方案 | v2.0 |
|------|--------|------|
| 筛选层级 | 单层 AI（70% 一刀切） | 三级漏斗（规则→AI→深度） |
| 数据来源 | 在线简历整页截图 | 分区域截图 + DOM文本 + 按需附件 |
| 匹配基准 | 无明确参照 | JD 结构化画像 |
| 评估结果 | 填表留存 | 分级自动触发动作 |
| 过程数据 | 无 | 时间戳 + 渠道 + 状态追踪 |
| 账号安全 | 未考虑 | 渐进下载 + 间隔 + 上限 |
| 回流机制 | 可能死循环 | 已筛选池去重 + 48h 超时 |
